Method 1: Python-script synthetic generation (messy and realistic simulation)
Source script: generate_synthetic_datasets.py

A) Ecommerce synthetic generation
Function: make_ecommerce_dataset(n_rows)
Main characteristics:
- Uses Faker plus seeded randomness for reproducibility (seed = 42).
- Generates realistic customer-level ecommerce behavior with demographic and behavioral features.
- Uses weighted categorical sampling (for gender, category, device, city-country pairs).
- Uses mixed/random distributions:
  - Beta distribution for age skew.
  - Poisson for order counts.
  - Normal noise for monetary values.
- Injects realistic data quality issues:
  - Missing values in non-protected columns (5-10 percent range).
  - Outliers in avg_order_value and total_spent.
  - Temporal inconsistencies (last_purchase_date before signup_date in a small fraction).
  - Category typos (for example: electrnics, grocery, fashon).
  - Minor text artifacts (double spaces, random inconsistencies).
- Shuffles rows before return.
Output file from main(): ecommerce_synthetic.csv
Typical row count range: random between 6200 and 9100.

B) Healthcare synthetic generation
Function: make_healthcare_dataset(n_rows)
Main characteristics:
- Uses seeded random generation and Faker identities.
- Generates patient records with mixed age populations (child/adult/senior) and diagnosis-driven severity/cost patterns.
- Uses weighted sampling for blood groups, diagnoses, and hospital type.
- Adds realistic structure and noise:
  - Severity and treatment cost correlated with diagnosis and length of stay.
  - Right-skewed treatment costs via lognormal base.
  - Insurance and hospital type cost multipliers.
- Injects data quality issues:
  - Missing values in non-protected columns.
  - Outliers in severity and treatment_cost.
  - Date inconsistencies (discharge before admission).
  - Diagnosis spelling drift (for example: Hypertensn, Diabtes, Migrane).
- Shuffles rows before return.
Output file from main(): healthcare_synthetic.csv
Typical row count range: random between 5700 and 8800.

Method 2: Fully synthetic AI-complete ecommerce generation
Source script: generate_ecommerce_ai_full.py
Function: make_ai_complete_ecommerce(n_rows=9000)
Main characteristics:
- Creates schema-aligned transaction data without external input.
- Uses handcrafted product catalog with category and price ranges.
- Generates invoice numbers, stock codes, product descriptions, quantities, dates, prices, customer IDs, and country values.
- Includes realism features:
  - Small cancellation-like and return-like behavior (C-prefixed invoices and negative quantities).
  - Country imbalance probabilities.
  - Price noise over product-specific ranges.
  - Small missingness in Description and CustomerID.
  - Row shuffling.
Output file from main(): ecommerce_ai_complete.csv
Default row count: 9000.

Method 3: Open-source ecommerce augmentation with synthetic rows
Source script: augment_ecommerce_with_ai.py
Main functions: load_open_source_ecommerce(), generate_augmented_rows(base_df, extra_rows=2500)
Main characteristics:
- Loads open-source UCI Online Retail dataset:
  https://archive.ics.uci.edu/ml/machine-learning-databases/00352/Online%20Retail.xlsx
- Validates required schema columns and coerces types.
- Generates additional synthetic rows aligned to source distribution:
  - Samples descriptions, stock codes, countries from source data with fallback values.
  - Samples/prices with noise and positivity constraints.
  - Samples quantity with return behavior.
  - Samples invoice dates across source time span.
  - Generates synthetic customer IDs.
- Appends source flag column dataset_source:
  - open_source_original
  - ai_augmented_rows
- Writes source attribution note to open_source_dataset_source.txt.
Output file from main(): ecommerce_open_source_augmented.csv
Synthetic rows added by default: 2500, appended to all original rows.
Population method (before and after results):
- Before population/augmentation:
  - Base dataset = cleaned open-source UCI Online Retail rows.
  - Row count = N (depends on current source read and cleaning outcome).
  - Column count before adding source flag = 8.
- Population step:
  - Adds extra_rows = 2500 synthetic rows by default.
  - Tags rows with dataset_source values: open_source_original or ai_augmented_rows.
- After population/augmentation:
  - Row count = N + 2500 (default behavior).
  - Column count after adding source flag = 9.
  - Final output file: ecommerce_open_source_augmented.csv.

Method 4: Ecommerce master orchestrator
Source script: generate_all_ecommerce_datasets.py
Main behavior:
- Calls make_ecommerce_dataset(...) and saves as ecommerce_python_script.csv.
- Calls make_ai_complete_ecommerce(n_rows=9000) and saves as ecommerce_ai_complete.csv.
- Calls augmentation pipeline, which saves ecommerce_open_source_augmented.csv.
Output files:
1. ecommerce_python_script.csv
2. ecommerce_ai_complete.csv
3. ecommerce_open_source_augmented.csv

Dataset Features (Count, Feature Names, and Summary)

1) ecommerce_synthetic.csv
- Feature count: 15
- Features:
  customer_id, customer_name, age, gender, city, country, signup_date,
  last_purchase_date, total_orders, avg_order_value, total_spent,
  preferred_category, device_used, is_returning_customer, customer_note
- Summary of features:
  - Customer identity/profile: customer_id, customer_name, age, gender, city, country.
  - Customer timeline: signup_date, last_purchase_date.
  - Purchase behavior: total_orders, preferred_category, device_used, is_returning_customer.
  - Monetary outcomes: avg_order_value, total_spent.
  - Unstructured text/context: customer_note.

2) healthcare_synthetic.csv
- Feature count: 14
- Features:
  patient_id, name, age, gender, blood_group, city, admission_date,
  discharge_date, diagnosis, severity_score, treatment_cost,
  insurance_covered, hospital_type, clinical_note
- Summary of features:
  - Patient identity/demographics: patient_id, name, age, gender, blood_group, city.
  - Hospitalization timeline: admission_date, discharge_date.
  - Clinical state: diagnosis, severity_score.
  - Financial/provider attributes: treatment_cost, insurance_covered, hospital_type.
  - Unstructured clinical context: clinical_note.

3) ecommerce_python_script.csv
- Feature count: 15
- Features:
  customer_id, customer_name, age, gender, city, country, signup_date,
  last_purchase_date, total_orders, avg_order_value, total_spent,
  preferred_category, device_used, is_returning_customer, customer_note
- Summary of features:
  - Same schema as ecommerce_synthetic.csv because it is produced by make_ecommerce_dataset(...)
    through the master orchestrator script.

4) ecommerce_ai_complete.csv
- Feature count: 8
- Features:
  InvoiceNo, StockCode, Description, Quantity, InvoiceDate,
  UnitPrice, CustomerID, Country
- Summary of features:
  - Transaction identifiers: InvoiceNo, StockCode.
  - Product and sales values: Description, Quantity, UnitPrice.
  - Time and customer context: InvoiceDate, CustomerID, Country.
  - Schema mirrors common online retail transactional structure.

5) ecommerce_open_source_augmented.csv
- Feature count: 9
- Features:
  InvoiceNo, StockCode, Description, Quantity, InvoiceDate,
  UnitPrice, CustomerID, Country, dataset_source
- Summary of features:
  - Core transaction schema: same 8 transaction columns as online retail format.
  - Provenance tracking: dataset_source indicates whether a row is open_source_original
    or ai_augmented_rows.
  - Suitable for experiments on mixed real + synthetic data behavior.
